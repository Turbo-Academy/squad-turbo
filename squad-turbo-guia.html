O que é um Squad

Pense num time de marketing — só que de IA.

Num squad de verdade tem estrategista, copywriter, diretor criativo, designer, gestor de tráfego, social media. Cada um com especialidade e autoridade sobre seu território. Squad Turbo reproduz isso dentro do Claude Code — 10 agentes que conversam entre si, passam tarefas, respeitam hierarquia e entregam peças prontas.

🧠 Especialistas, não genérico

Cada agente tem um papel único. Copywriter escreve copy. Diretor criativo dá direção visual. Designer executa. Ninguém faz o trabalho do outro.

🔗 Hierarquia real

O estrategista orquestra. O pesquisador segura a fundação. Copy e visual só rodam depois do briefing pronto. Zero peça saindo sem base.

📦 Self-contained

A v2.1 é totalmente independente. Baixou a pasta, colou em ~/.claude/squads/, tá rodando. Não depende de nenhuma outra instalação.

⚡ Metodologia da Turbo × Projeto do cliente

A metodologia (Lançamento Pago Semanal, Método 5+1, Funil 8) é o que a Turbo Academy te ensinou e está embarcada no squad. O projeto onde o squad opera é o do seu cliente (ou o seu, se você é o próprio expert). Na primeira execução em cada projeto, o @pesquisador-turbo monta a fundação daquele cliente (avatar, voz, oferta) em 00-fundacao/. É aí que a metodologia vira entrega concreta.

Os 10 agentes

Quem trabalha no seu squad

Cada agente tem um tier (hierarquia) e uma autoridade (o que ele pode e não pode mexer).

Tier 0 · Orquestrador

Estrategista Turbo

@estrategista-turbo

Ponto de entrada. Recebe seu pedido, faz diagnóstico (TYPE/SCOPE/STAGE), checa se a fundação existe, monta o time e delega. Nunca executa copy ou visual.

Tier 0 · Fundação do Projeto

Pesquisador Turbo

@pesquisador-turbo

Consome material do seu projeto (aulas, docs, copy atual, vídeos) e consolida 6 dossiês em 00-fundacao/: voz, avatar, oferta, briefing, referências, inventário. Autoridade exclusiva sobre essa pasta.

Tier 0 · Mercado Externo

Pesquisador de Mercado

@pesquisador-mercado-turbo

Pesquisa web, concorrência, benchmarks, Meta Ad Library, YouTube. Autoridade exclusiva sobre 02-mercado/.

Tier 1 · Copy

Copywriter Turbo

@copywriter-turbo

Hub de copy: páginas, anúncios, reels, email, WhatsApp, VSL, scripts de workshop, Método 5+1. Lê a fundação antes de escrever.

Tier 1 · Direção Visual

Diretor Criativo Turbo

@diretor-criativo-turbo

Define paleta, tipografia, composição, moodboard. Recebe copy aprovada, passa brief pro designer. Não toca arquivo — só coordena.

Tier 2 · Execução Visual

Designer Turbo

@designer-turbo

Produz os arquivos finais: criativos, slides, brandbook, banners, landing pages. Trabalha sob brief do diretor criativo.

Tier 1 · Social

Social Turbo

@social-turbo

Conteúdo orgânico, reels, carrosséis, stories. Ritmo de perfil e calendário editorial.

Tier 2 · Mídia

Tráfego Turbo

@trafego-turbo

Campanhas Meta e Google. Estrutura de contas, testes, otimização.

Tier 2 · Automação

Automação Turbo

@automacao-turbo

n8n, ManyChat, mensageria de lançamento, fluxos WhatsApp, integrações.

Tier 3 · Pós-venda

CS Turbo

@cs-turbo

Customer Success, onboarding de aluno, retenção, comunicação pós-matrícula.

Como funciona por dentro

O fluxo de uma peça do começo ao fim

Quando você pede qualquer coisa, esse é o caminho que o squad percorre:

VOCÊ         →  "preciso de uma página de vendas pro meu workshop"
     ↓
@estrategista-turbo   →  Diagnostica: TYPE=página, SCOPE=workshop, STAGE=venda
     ↓                   →  Checa 00-fundacao/
     ↓
  FUNDAÇÃO OK?
     ├── NÃO →  @pesquisador-turbo  →  gera 6 dossiês  →  LOCKED
     └── SIM ↓
@copywriter-turbo     →  lê 00-fundacao/  →  escreve copy
     ↓
  VOCÊ APROVA A COPY
     ↓
@diretor-criativo-turbo  →  define direção visual  →  brief
     ↓
@designer-turbo     →  executa HTML/arquivos finais
     ↓
/page-optimizer    →  otimiza pra plataforma de destino
     ↓
  PEÇA ENTREGUE ✓

⚠️ Regra de ouro: Fundação antes de execução

Nenhuma peça final sai sem 00-fundacao/ LOCKED. Se faltar, o fluxo para e vai pro pesquisador. Isso não é burocracia — é o que separa copy genérica com cara de IA de copy no tom real do seu expert.

Como usar na prática

3 formas de chamar o squad

1

Invocação direta por agente

Chama o agente pelo handle. Use quando sabe exatamente quem precisa.

# no Claude Code, digita:
@copywriter-turbo escreve 5 headlines pro meu workshop no tom do expert

@designer-turbo gera um criativo 1080x1350 com paleta do brandbook
2

Pela porta da frente (recomendado)

Pede normal. O @estrategista-turbo diagnostica e delega pra quem precisa.

# exemplo:
preciso lançar meu workshop semana que vem, me ajuda a montar o funil completo

Ele vai checar fundação, escolher o funil certo (Método 5+1, Funil 8, Workshop), montar o time e orquestrar.

3

Via slash commands (skills)

O squad vem com 20 skills prontas pra tarefas específicas.

  • /designer-senior — landing pages HTML com Protocolo Anti-IA
  • /gerador-instagram — carrosséis do Instagram
  • /criador-reels — roteiros de reels
  • /page-optimizer — otimização pós-aprovação
  • /lancamento-pago-semanal — fluxo completo do método

Instalação

Como baixar e instalar

O Squad Turbo é distribuído pelo GitHub oficial da Turbo Academy. Duas formas de instalar:

Opção A — Via Git (recomendado)

Se você tem git instalado, é a forma mais rápida — e o único jeito de receber atualizações com um comando só.

# 1. Cria a pasta de squads (se não existir)
mkdir -p ~/.claude/squads

# 2. Clona o repositório
cd ~/.claude/squads
git clone https://github.com/Turbo-Academy/squad-turbo.git

# 3. Verifica que funcionou
ls ~/.claude/squads/squad-turbo/
# deve listar: agents/ skills/ core/ squad.yaml ...

Opção B — Download ZIP

Sem git? Sem problema.

  1. Acessa github.com/Turbo-Academy/squad-turbo
  2. Clica no botão verde Code → Download ZIP
  3. Descompacta o arquivo
  4. Renomeia a pasta pra squad-turbo (remove o -main do fim)
  5. Move pra ~/.claude/squads/squad-turbo/

Atualizando o squad

O squad é atualizado com frequência pela equipe da Turbo Academy — novos agentes, skills, correções, melhorias no método. Atualize sempre que começar um projeto novo.

# se instalou via git (1 comando):
cd ~/.claude/squads/squad-turbo
git pull

# se instalou via ZIP:
# baixa o ZIP novo e substitui a pasta inteira

Verificando instalação

Abre o Claude Code numa pasta qualquer e digita:

@estrategista-turbo oi, lista os agentes do squad

Se ele responder listando os 10 agentes, tá tudo certo. Se disser que não conhece o agente, reinicia o Claude Code e confere o caminho ~/.claude/squads/squad-turbo/.

O que você pode e não pode fazer

Regras de uso

🚫 Não modifique os arquivos do squad

Agentes, skills, constitution, templates e frameworks são propriedade da Turbo Academy e recebem atualizações constantes do time do Leo — ajustes no método, novos agentes, correções, evolução das skills. Se você editar qualquer arquivo da pasta ~/.claude/squads/squad-turbo/, na próxima atualização (git pull) você vai ter conflito ou perder suas alterações. Trate a pasta como somente-leitura.

✅ O que VOCÊ manda:

Tudo que está dentro da pasta do SEU projeto: 00-fundacao/ (seu avatar, voz, oferta), 02-mercado/, copy final, páginas, criativos, briefings. O squad é a ferramenta. O conteúdo do seu projeto é seu.

Se precisar customizar algo

  • Quer um comportamento diferente? Crie um agente derivado no SEU projeto (ex: meu-projeto/.claude/agents/copywriter-custom.md) em vez de editar o original.
  • Quer uma skill nova? Mesma lógica — crie dentro do seu projeto, não dentro do squad.
  • Encontrou bug ou tem sugestão? Manda pro canal oficial de suporte da Turbo Academy. Quem atualiza o squad é o time do Leo.

Regras inegociáveis do squad

  • Fundação antes de execução — sem 00-fundacao/ LOCKED, nenhuma peça final sai
  • Zero invenção — nunca fabricar depoimentos, métricas ou resultados
  • Tom do expert — copy no tom do expert do seu projeto (de voz.md), nunca genérico
  • Protocolo Anti-IA visual — zero elemento com cara de IA em qualquer peça visual
  • Uma variável por teste — mudou duas coisas ao mesmo tempo, não sabe o que funcionou
  • Português sempre
Dúvidas comuns

FAQ

Preciso pagar alguma coisa além do Claude?

Não. O Squad Turbo é distribuído gratuitamente pelos canais oficiais da Turbo Academy pros alunos. Você só precisa ter o Claude Code instalado (plano Pro ou superior recomendado pra uso intenso).

Qual a diferença entre "agente" e "skill"?

Agente = trabalhador autônomo com papel e autoridade (ex: @copywriter-turbo). Você conversa com ele.
Skill = prompt especializado invocado por slash command (ex: /gerador-instagram). É uma ferramenta que os agentes (ou você) podem usar.

Posso usar o Squad Turbo no projeto do meu cliente?

Sim — é exatamente pra isso. O squad carrega a metodologia da Turbo e aplica no projeto que você estiver operando. Na primeira vez em cada cliente, o @pesquisador-turbo pede o material bruto (aulas, páginas atuais, copy, lives, VoC) e consolida a 00-fundacao/ daquele cliente específico. A partir daí, todos os agentes produzem no tom e no contexto corretos daquele projeto. Você pode atender múltiplos clientes, cada um na sua própria pasta.

O que é "fundação LOCKED"?

É quando os 6 dossiês em 00-fundacao/ (voz, avatar, oferta, briefing, referências, inventário) foram consolidados pelo @pesquisador-turbo e validados por você. Sem isso, nenhuma peça final sai — porque sem fundação a copy vira genérica com cara de IA.

O squad funciona offline?

A estrutura fica na sua máquina (offline), mas os agentes rodam no Claude, que precisa de internet. O @pesquisador-mercado-turbo depende de WebSearch pra funcionar.

Como atualizo pra uma versão nova?

Se instalou via git, git pull dentro da pasta. Se foi ZIP, baixa a versão nova e substitui. O time do Leo atualiza com frequência — vale atualizar toda vez que começar um projeto novo.

Posso modificar os agentes ou as skills?

Não. A pasta ~/.claude/squads/squad-turbo/ é mantida pelo time do Leo e recebe atualizações constantes. Qualquer edição sua nos arquivos do squad será perdida no próximo git pull (ou dará conflito). Se quiser customizar comportamento, crie agentes/skills derivados dentro do SEU projeto — nunca dentro da pasta do squad. Tem sugestão ou bug? Manda no canal de suporte oficial.

O que fazer se o agente não responder?

Três checks: (1) pasta está em ~/.claude/squads/squad-turbo/? (2) reinicia o Claude Code; (3) confere se o nome do agente está com @ na frente e o handle exato (ex: @copywriter-turbo, não @copy). Se persistir, manda pro suporte.

Meus arquivos ficam dentro da pasta do squad?

Não. Seus projetos ficam onde você quiser (ex: ~/Projetos/lancamento-cliente-x/). O squad é instalado uma vez em ~/.claude/squads/squad-turbo/ e serve todos os seus projetos. Cada projeto tem seu próprio 00-fundacao/, 02-mercado/, copy, páginas, etc.
