# Framework Maxxima — 16 Passos Invariantes para Páginas Low Ticket

Extraído da engenharia reversa de 3 páginas de Ricardo Maxxima, todas com conversão comprovada em tráfego frio para produtos tangíveis low-ticket.

Fontes analisadas:
1. Kit Copy e Página de Vendas (12x R$9 = R$87, âncora R$707)
2. Cartilha do Tráfego Pago Low Ticket (5x R$8,19 = R$37, âncora R$314)
3. Biblioteca de Criativos com ROI 2+ (R$17/mês assinatura, âncora R$600)

---

## Quando usar este framework

Usar como blueprint quando o produto é **LOW TICKET TANGÍVEL** (kit, cartilha, protocolo, biblioteca) com preço entre R$17–97, vendido para tráfego frio. Para outros tipos de página low-ticket (evento pago, desafio), os exemplos Patricia/Mapa/Indulto/Dentística podem ser mais apropriados.

Este framework **NÃO substitui** o diagnóstico Schwartz × sofisticação. Ele define a **ESTRUTURA**. O diagnóstico define a **PROFUNDIDADE** de cada passo.

---

## Os 16 Passos Invariantes

### Passo 1 — Headline Imperativa com Resultado

Verbo de ação direto no imperativo. "COPIE o Kit Completo de Copy e Página de Vendas", "COPIE e Aplique a Cartilha". O avatar lê e sabe exatamente o que vai ter. Nunca interrogativa ("Como aumentar..."). Nunca genérica ("Transforme sua vida").

**Diferença dos 7 Padrões Universais:** Os padrões universais (Padrão 1) dizem que a promessa é sobre a **PESSOA**. No framework Maxxima, a promessa também é sobre a pessoa **MAS** o formato é imperativo — o avatar é colocado como agente da ação. "COPIE" não é uma promessa passiva, é uma instrução de ação.

### Passo 2 — 3 Bullets de Velocidade

Imediatamente abaixo da headline. Eliminam objeções em 3 segundos. Formato: "✅ Sem experiência prévia / ✅ Sem ferramentas caras / ✅ Resultado em 24h". Cada bullet mata uma objeção específica.

Não existe equivalente nos 7 Padrões Universais — é uma adição do framework Maxxima.

### Passo 3 — Visual do Produto em Uso

Mockup do produto sendo usado/manuseado. O "dedo no criativo" — expert segurando ou apontando o produto tangível. Se o produto é digital, criar mockup 3D (capa de cartilha, caixa de kit, etc.).

**Relação com Padrão 2:** O Padrão 2 diz "Dobra 2 é sempre identificação ou prova". No Maxxima, a Dobra 2 (visual) **É** prova — prova visual de que o produto existe e é real.

### Passo 4 — Prova Social Numérica Imediata

Número grande + específico na primeira metade da página. "32.487 pessoas já usam", "2.400 criativos testados". Nunca genérico ("milhares"). Sempre número quebrado (mais crível que número redondo).

**Relação com Padrão 7 (Peça 7 da Promessa):** A Dobra 1 já pode conter prova de escala, mas no Maxxima é uma seção separada com destaque visual próprio.

### Passo 5 — "EU SEI..." + Espelho da Dor em Balões de Chat

Formato de mensagens simulando chat real. "Eu sei que você já tentou..." / "Eu sei que você sente que...". Cada balão = uma dor específica. Máximo 3–5 dores.

**Relação com Padrão 3:** É a transferência de culpa do Padrão 3, mas com formato visual específico (chat bubbles). A profundidade segue a mesma regra: mais vergonha = mais dores.

### Passo 6 — Frase de Transição + Micro-compromisso

"E se eu te mostrasse que existe um caminho mais simples?" / "A verdade é que o problema não é você." Ponte entre a dor e a solução.

### Passo 7 — Passo a Passo de Uso

3–5 passos mostrando como usar o produto. Visual com ícones/números. Remove a objeção "será que vou saber usar?"

### Passo 8 — Stack de Valor com Mockups Individuais

Cada componente com mockup visual próprio. Produto principal + bônus. Cada mockup aumenta percepção de valor tangível.

**Diferença dos 7 Padrões:** O Padrão 5 fala de preço enquadrado por comparação. No Maxxima, o enquadramento começa **ANTES** com o stack visual — o avatar vê o volume antes de ver o preço.

### Passo 9 — "AINDA NÃO ACABOU!"

Gatilho de surpresa e acúmulo. Aparece depois do stack principal, antes de bônus extras. Amplifica a percepção de que o preço é irrisório.

Não existe nos 7 Padrões — é específico do framework Maxxima.

### Passo 10 — "Para Quem É"

Lista de perfis beneficiados. Auto-seleção. Opcional: "Para quem NÃO é" (exclusão que cria desejo).

**Relação com Dobras Finais:** Já existe na skill como "Qualificação + CTA Final" (dobras-finais.md). No Maxxima, aparece **antes** do preço, não depois.

### Passo 11 — Autoridade por Associação

Logos, veículos, prints de resultados de clientes com dados visíveis. **NÃO é a bio do expert** — é prova de mercado.

**Diferença:** Na skill atual (Padrão 4), o expert aparece tarde ou não aparece. No Maxxima, a **AUTORIDADE POR ASSOCIAÇÃO** aparece antes da bio. São coisas diferentes — associação = logos e provas externas; bio = quem é o expert.

### Passo 12 — Ancoragem de Preço

Soma de componentes → valor total tachado → preço real. **SEMPRE numérica**. "De ~~R$707~~ por apenas 12x R$9". Relação direta com Padrão 5.

### Passo 13 — Bifurcação de 2 Opções

"Fechar esta página" (consequências) vs. "Garantir acesso" (benefícios + preço). Já existe na skill como "Dilema binário no fechamento" (dobras-finais.md).

### Passo 14 — Bloco de Autoridade do Expert

Bio curta, credenciais, foto. **DEPOIS** do preço. Alinhado com Padrão 4.

### Passo 15 — CTA Final

Botão com ação + resultado. Selos de segurança. Garantia incondicional.

### Passo 16 — Footer

© + Termos + Privacidade + Disclaimer.

## Análise de Gap — Exemplo Marlem Moraes

**Página:** marlemmoraes.com.br/protocolo-cintura-definida-manual/  
**Produto:** Protocolo Cintura Definida Manual (R$62 / 8x R$9,40)  
**Nicho:** Estética corporal B2B (esteticistas/massoterapeutas)

**Aderência ao framework:** 6/16 presentes, 5 adaptados com fraquezas, 5 ausentes = **37,5%**

### Presentes ✅
- Transferência de culpa
- Para quem é/não é
- Bifurcação emocional
- Autoridade do expert
- CTA final
- Footer

### Adaptados com Fraquezas ⚠️
- **Headline:** Explicativa ao invés de imperativa
- **Espelho da dor:** Cards ao invés de chat bubbles
- **Passo a passo:** Sem visual do produto
- **Stack de bônus:** Sem mockups individuais
- **Ancoragem:** Implícita ao invés de numérica

### Ausentes ❌
- Bullets de velocidade
- Prova social numérica imediata (na headline)
- "AINDA NÃO ACABOU!"
- Autoridade por associação (logos/veículos)
- Visual do produto tangível sendo manuseado

### Erros Operacionais
- Inconsistência de preço (R$62 no corpo vs R$35 no FAQ)
- VSL de ~3.000px bloqueando scroll

### Lição
Mesmo páginas com copy competente perdem conversão quando ignoram os elementos estruturais do framework — especialmente:
1. **Tangibilidade visual** (product in hand)
2. **Ancoragem numérica** (de → por)
3. **Prova social acima do fold** (número quebrado, específico)

## Como Aplicar Este Framework

1. **Diagnóstico primeiro:** Use `references/diagnostico-pre-pagina.md` para entender avatar, Schwartz e sofisticação.
2. **Escolha a profundidade:** Se o nicho tem alta vergonha (ex.: emagrecimento), o Passo 5 (espelho da dor) ganha 5–7 balões. Se é técnico, reduz para 2–3.
3. **Visual é obrigatório:** Produtos tangíveis precisam do Passo 3 (visual em uso) + Passo 8 (mockups dos componentes). Sem isso, a conversão cai 30–40%.
4. **Sequência é rígida:** Diferente dos 7 Padrões (que permite reordenação), os 16 passos Maxxima funcionam porque obedecem essa ordem específica. Testar variações de ordem reduz ROI.
5. **Teste uma coisa por vez:** Se quer testar "Passo 2 (bullets) vs sem bullets", isole essa mudança. Não mude headline + bullets + visual ao mesmo tempo.
